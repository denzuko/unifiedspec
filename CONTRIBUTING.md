# Contributing to the Unified Design System

## What this is

The Unified Design System is a specification, not a software package.
Contributions follow an editorial process, not a pull request merge process.

## Two types of contribution

### Errata

An erratum corrects a factual error, typo, or technical inaccuracy in the
current edition. Errata do not change normative requirements.

**To report an erratum:** open an issue using the **Errata report** template.

Accepted errata are applied to the current draft and noted in the commit
history. They do not advance the edition number.

### Amendments

An amendment proposes a new capability, a behaviour change, or an addition
to the specification. Amendments may change conformance criteria.

**To propose an amendment:** open an issue using the **Amendment proposal**
template.

Amendments are held for review body ratification. No amendment is
incorporated until the review body has ratified it and the 60-day public
comment period has closed without blocking objections.

## Review body

The review body is not yet formally constituted. Until it is:

- Da Planet Security holds editorial authority over the draft.
- Amendment proposals are tracked but not ratified.
- Errata are applied at editorial discretion.

## Style

All normative text uses RFC 2119 terms: MUST, MUST NOT, SHOULD, SHOULD NOT,
MAY, OPTIONAL, REQUIRED, RECOMMENDED.

Prose: practitioner-to-practitioner. Present tense. Active voice. Sentence
case for headings and labels. No rhetorical questions.
